# TODO 寫作與管理規範

(TODO Writing & Management Specification)

本文件定義了專案中 `TODO.md` 的寫作格式、優先級定義與維護流程，旨在確保開發目標清晰且易於追蹤。

---

## 1. 檔案位置與結構

- **檔案位置**：專案根目錄 `/TODO.md`
- **基本結構**：

```markdown
# 待辦事項 (TODO)

> 最後更新：YYYY-MM-DD (Context/Workflow)

## 🔴 P0: 立即執行 (Critical / High ROI)

**目標**：[簡述本階段核心目標]

- [ ] 待辦事項 1
- [ ] 待辦事項 2

---

## 🟡 P1: 技術清理 (High Value / Low Risk)

**目標**：[簡述本階段核心目標]

- [ ] 待辦事項 A

---

## 🟢 P2: 核心功能 (Must Have)

...

---

## ⚪ P3: 未來規劃 / 待定

...

---

## 🏁 已完成歸檔 (Archived)

- [x] 已完成事項 (Completed: YYYY-MM-DD)
```

---

## 2. 優先級定義 (Priority Levels)

| 等級   | Emoji | 定義         | 適用情境                                                                                                        |
| :----- | :---- | :----------- | :-------------------------------------------------------------------------------------------------------------- |
| **P0** | 🔴     | **立即執行** | 阻礙開發的 Bug、高視覺回報 (High Visual ROI) 的修復、Production Critical 項目。**此區塊應保持極簡，專注當下。** |
| **P1** | 🟡     | **技術清理** | 高價值但低風險的任務 (High Value/Low Risk)、重構、程式碼品質提升、準備工作。                                    |
| **P2** | 🟢     | **核心功能** | 產品必須具備的功能 (Must-Have)，如 PWA 離線支援、關鍵用戶流程。                                                 |
| **P3** | ⚪     | **未來規劃** | 待定功能、優化想法 (Nice-to-Have)、長期規劃。                                                                   |

---

## 3. 寫作格式 (Writing Format)

### 3.1 目標描述 (Goal)

每個優先級區塊下，建議先用 `**目標**：...` 簡述該區塊當前的聚合目標。

> 範例：`**目標**：修復視覺斷層，讓現有功能完整化。`

### 3.2 待辦事項 (Items)

- 使用 `- [ ]` 列表。
- 文字應簡潔明確，最好包含動詞 (Review, Fix, Add, Refactor)。
- 若有相關檔案路徑，建議使用 Backticks 標示 (如 `src/App.jsx`)。

### 3.3 歸檔格式 (Archiving)

82: 當項目完成 (`[x]`) 並經過 Backlog Grooming 後，應移動至歸檔區的**最上方** (Reverse Chronological)。
83: 
84: - **格式**：`- [x] 原標題 ✅ (Completed: YYYY-MM-DD)`
85: - **說明**：必須加上完成日期。最新的完成記錄應置於列表頂端，以利快速檢視近期產出。

89: ## 4. 維護流程 (Maintenance Workflow)
90: 
91: 1.  **Check (檢查)**: 每次提交程式碼前，檢查 `TODO.md` 中的 `[x]` 項目。
92: 2.  **Archive (歸檔)**: 將已完成項目移至 `Archived` 區塊的**最上方** (置頂)，標註日期，並清空原 P0-P2 區塊 (若已無項目)。
3.  **Update (更新)**: 更新頂部的「最後更新」日期與 Context (如 `(Backlog Grooming)` 或 `(Advisor Sync)`)。
4.  **Promote (晉升)**: 當 P0 清空時，從 P1/P2 選取最重要項目晉升至 P0。
