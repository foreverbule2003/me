---
description: 快速捕捉靈感或任務至 TODO.md
---

1. **讀取現有清單**
   - 使用 `view_file` 讀取專案根目錄下的 `TODO.md`。

2. **分析與分類**
   - 分析使用者輸入的內容 `[想法]`。
   - 根據緊急程度與類型判斷歸屬：
     - **P0**: 嚴重 Bug、立即阻礙開發的問題。
     - **P1**: 技術債、架構優化、DevOps。
     - **P2**: 新功能、UI 優化、內容擴充。
     - **Inbox**: 如果無法確定，或屬於模糊的未來靈感，放入 P2 底部或建立新的 Inbox 區塊。

3. **寫入更新**
   - 使用 `replace_file_content` 或 `multi_replace_file_content` 將項目插入對應章節的列表底部。
   - 格式應為：`- [ ] [想法] <!-- captured: [日期] -->`

4. **確認回報**
   - 簡短回報：「已捕捉 '[想法]' 至 [類別]。」
