---
description: 以資深架構師角度審查專案架構、安全性與最佳實踐
---

執行此 workflow 時，請扮演**資深軟體架構師**，對專案進行全面審查。

## 審查清單

### 1. 架構一致性
- [ ] 是否存在技術棧混用/不一致? (例如部分頁面用 React，部分用 Vanilla JS)
- [ ] 元件設計是否遵循 DRY 原則?
- [ ] 資料結構是否統一?
- [ ] 命名慣例是否一致?

### 2. 安全性
- [ ] API Keys 是否有適當的限制? (HTTP Referrer, API restrictions)
- [ ] Firestore/RTDB 規則是否遵循最小權限原則?
- [ ] 是否有敏感資訊暴露風險?
- [ ] 認證/授權流程是否完整?

### 3. 效能與可維護性
- [ ] 是否有重複的程式碼可以抽取為共用元件?
- [ ] 資源載入是否優化? (lazy loading, code splitting)
- [ ] 是否有潛在的效能瓶頸?

### 4. 文件完整性
- [ ] README.md 是否與實際功能同步?
- [ ] 各功能文件 (FEATURES.md, COMPONENTS.md, SITEMAP.md) 是否完整?
- [ ] 是否有缺少的 spec.md 或技術文件?

### 5. 待辦與技術債
- [ ] 是否有標記為 TODO/FIXME 的項目?
- [ ] 是否有已棄用的程式碼?
- [ ] 是否有 Hardcoded 的值應該提取為設定?

## 輸出格式

請產出一份 **ARCHITECTURAL_REVIEW.md** 報告，包含：
1. 執行摘要 (1-2 句話總結專案健康度)
2. 發現的問題 (依嚴重程度排序：🔴 高 / 🟡 中 / 🟢 低)
3. 具體建議與修復方案
4. 值得肯定的設計決策
