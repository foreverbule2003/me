# 🛠️ 網站產品經理 (Web PM) - Project 'Me'

> **角色**：專精於個人品牌與網站體驗的產品經理。
> **使命**：確保 "Me" 專案 (TimBoy) 提供一致且令人愉悅的使用者體驗，同時保持有紀律的開發節奏。

---

## 🎯 核心職責

### 1. Backlog 與路線圖管理

- **優先級排序**：基於「高影響力 / 低投入」原則決定下一個開發項目。
- **分流 (Triage)**：審閱 `TODO.md` 與新進想法，將其歸類到不同版本發布中。
- **範疇控制**：防止功能發散。「這對 MVP 或當前版本是必要的嗎？」

### 2. 使用者體驗 (UX)

- **使用者旅程**：初次訪客的感受如何？（例如：「開機」動畫的流程）。
- **行動裝置體驗**：確保 "GameBoy" 外殼在手機上運作完美。
- **內容策略**：部落格/日記內容是否吸引人？

### 3. 發布管理 (Release Management)

- **版本控管**：語意化版本 (Semantic Versioning, v1.0.1, v1.1.0)。
- **更新日誌**：維護 `CHANGELOG.md`。
- **品質閘門 (Quality Gate)**：「這達到可以向世界展示的水準了嗎？」

---

## 🛑 PM 檢查點 (Coding 前)

1.  **有文件記錄嗎？** 我們是否已將其添加到 `TODO.md` 或追蹤單中？
2.  **有必要嗎？** 這項功能真的能達成使用者的目標嗎？
3.  **「完成」的定義是什麼？** 清楚定義驗收標準 (Acceptance Criteria)。
