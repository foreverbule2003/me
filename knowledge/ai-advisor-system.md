# 🤖 AI 顧問系統 (AI Advisor System)

這是本專案的核心架構之一：一個「層級式 AI 專家團隊」，用於協助開發決策、架構設計與品質管理。

---

## 🧠 設計理念

**問題**：單一 AI 視角容易有盲點，且不同任務（如前端 UI vs. 後端架構）需要不同專長。

**解決方案**：建立多角色協作系統，各司其職，最後由超級角色整合。

---

## 👥 專家團隊

| 角色       | 檔案                   | 專長                                 |
| ---------- | ---------------------- | ------------------------------------ |
| 🎯 接待員   | `receptionist-prompt`  | 分診、流程引導                       |
| 🛠️ PM       | `web-pm-prompt`        | 進度管理、Backlog 排序、產品規格審核 |
| 💻 CTO      | `cto-prompt`           | 技術架構、性能優化、技術債管理       |
| 🎨 設師師   | `designer-prompt`      | UI/UX 設計、視覺美感                 |
| 👑 CSO      | `cso-prompt`           | 初步戰略決策與優先順序               |
| 💼 CIO      | `cio-prompt`           | 第一級第二意見 (風險預警)            |
| 🎭 CRO      | `cro-prompt`           | 第二級全域第二意見 (決策挑戰)        |
| 🧘 思維教練 | `thought-coach-prompt` | 想法收斂、結構化思考                 |

---

## 🔗 協作流程

```
/advisor-sync 或 /help
         │
         ▼
┌─────────────────────────┐
│  接待員 (分診)          │ ← 不確定時先來這
└─────────────────────────┘
         │
         ▼
┌─────────────────────────┐
│  PM + 設師師            │ ← 產品與視覺定義
└─────────────────────────┘
         │
         ▼
┌─────────────────────────┐
│  CTO (技術評估)         │ ← 技術實作與架構檢核
└─────────────────────────┘
         │
         ▼
┌─────────────────────────┐
│  CSO (初步決策)         │ ← 整合觀點定調方向
└─────────────────────────┘
         │
         ▼
┌─────────────────────────┐
│  CIO + CRO (加壓測試)    │ ← 第二意見、風險預警
└─────────────────────────┘
         │
         ▼
      最終共識
```

---

## 🎯 使用方式

| 情境         | 指令               |
| ------------ | ------------------ |
| 不確定該找誰 | `/help [描述問題]` |
| 全面專案診斷 | `/advisor-sync`    |
| 工作流檢視   | `/doc-check`       |
